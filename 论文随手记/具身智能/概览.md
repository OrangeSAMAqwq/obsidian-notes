# 具身智能（Embodied Intelligence）2026 完整路线概览  
> 总览 01–06 所有论文、技术范式与研究脉络  
> 用于构建个人知识体系 & 博士研究方向

---

# 📌 0. 具身智能的核心问题（哲学起点）

> [!summary] 具身智能的目标  
> **让机器体在真实世界中感知、理解、预测、规划、执行动作。**

机器人要同时解决：

- 视觉（Perception）  
- 世界模型（World Modeling）  
- 决策（Decision Making）  
- 控制（Control）  
- 语言理解（Language Understanding）  
- 泛化（Generalization）  

而 01→06 是从低层控制到 AGI 的完整进化序列。

---

# 01 — Reinforcement Learning Foundations（强化学习基础）

> [!summary] **目标：让机器人学会“如何选择动作”**

## 🎯 代表论文  
- **SAC（2018）**  
- **QT-Opt（2018）**

## 🧠 技术范式  
- 最大熵 RL  
- Off-policy actor-critic  
- Q-learning with continuous actions  
- Distributed Q-learning (QT-Opt)

## 📌 本质能力  
- 稳定的连续控制  
- 工业级 RL (Google Robotics)

## → 为什么进入下一阶段？  
RL 在真实世界太贵、sample-inefficient，因此需要：

> **让机器人学会预测未来 → 世界模型（World Models）**

---

# 02 — World Models（学习世界的潜在动力学）

> [!summary] **目标：让机器人能“在脑海里想象未来”**

## 🎯 代表论文  
- **PlaNet（2019）**  
- **Dreamer 系列（2019–2023）**

## 🧠 技术范式  
- RSSM（Recurrent State Space Model）  
- latent imagination rollouts  
- model-based control  
- behavior learning from latent dynamics

## 📌 本质能力  
- 机器人能通过 latent model 预测未来  
- 大幅减少真实环境交互需求  

## → 为什么进入下一阶段？  
世界模型强大，但效果仍受限于：

- 单一策略  
- 缺乏序列结构  
- 无法表达复杂动作分布  

于是进入：

> **GPT 风格的序列建模（Sequence Modeling for Control）**

---

# 03 — Sequence Modeling for Control（使用 Transformer 做控制）

> [!summary] **目标：把机器人控制问题变成“语言模型生成序列”**

## 🎯 代表论文  
- **Decision Transformer（2021）**  
- **Trajectory Transformer（2021）**  
- **CQL（2020, Offline RL）**

## 🧠 技术范式  
- RL → sequence modeling paradigm  
- 用 Transformer 生成：state, action, RTG  
- 行为克隆的“GPT 化”  
- Offline dataset → 动作预测

## 📌 本质能力  
- 机器人行为 = 序列 → 可被 transformer 捕捉  
- 利用 offline dataset，无需在线 RL  
- 打开“机器人 GPT” 方向的大门

## → 为什么进入下一阶段？  
Transformer 难以直接表达连续动作分布 ⇒  
需要更强的 **生成式模型**：

> **Diffusion Policy（生成动作轨迹）**

---

# 04 — Generative Action Policies（扩散模型用于机器人行为）

> [!summary] **目标：用 Diffusion 生成多模态动作轨迹（更稳定、更精准）**

## 🎯 代表论文  
- **Diffusion Policy（2023）**  
- **3D Diffuser Actor（2024）**  
- **3D Diffusion Policy（2024）**

## 🧠 技术范式  
- 条件扩散模型 (Conditional Diffusion)  
- 动作轨迹作为信号（trajectory as signal）  
- 2D → 3D 输入（RGB → 点云/voxel）  
- 生成未来 H-step actions

## 📌 本质能力  
- 强杂讯鲁棒性  
- 强生成性（multi-modal action distribution）  
- 高精度 3D 操控  
- 控制能力被显著提升

## → 为什么进入下一阶段？  
Diffusion Policy 强，但仍缺乏：

- 任务理解（language）  
- 物体中心抽象（object-centric）  
- 碰撞感知（collision-aware）  
- 泛化能力  

于是出现：

> **语言 + 物体中心 + 安全操控的扩散模型**

---

# 05 — Object-Centric & Language-Guided Generative Control  

> [!summary] **目标：让机器人依据语言指令，在 3D 场景中操作特定对象，并避免碰撞**

## 🎯 代表论文  
- **Language-Guided Object-Centric Diffusion Policy（2024）**  
- **VoxPoser（2023）**  
- **Object Diffusion Variants（2024）**

## 🧠 技术范式  
- Point cloud → object-centric representation  
- Language grounding via cross-attention  
- Collision-aware diffusion  
- Semantic manipulation

## 📌 本质能力  
- 机器人理解任务（language）  
- 识别目标（object-centric）  
- 规划安全轨迹（collision-free）  
- 泛化到新物体 / 新任务

## → 为什么进入下一阶段？  
机器人已经能理解任务 + 操作对象，那么下一步就是让机器人具有：

> **世界知识 + 推理能力 + 跨任务泛化 → Foundation Robot Models**

---

# 06 — Foundation Robot Models（VLA：Vision-Language-Action 大模型）

> [!summary] **目标：让机器人具备类似 GPT 的世界知识与泛化行动能力**

## 🎯 代表论文  
- **RoboCat（2023）**  
- **RT-1（2022）**  
- **RT-2（2023）**

## 🧠 技术范式  
- Transformer backbone  
- Visual tokens + Language tokens + Action tokens（VLA）  
- Scaling laws in robotics  
- Web knowledge → robot action (RT-2)  
- Multi-embodiment learning（RoboCat）

## 📌 本质能力  
- 使用语言理解任务  
- 使用 VLM 的世界知识推理  
- Zero-shot manipulation  
- 跨物体、跨场景、跨任务泛化  
- 跨机器人迁移（RoboCat）

## → 时代意义  
> **RT-2 是“机器人具身智能向 AGI 对齐”的第一次技术性落地。**

# 📌 全路线图（最终总览）

```markdown
01 Reinforcement Learning  
   └─ SAC → QT-Opt
        ↓  
02 World Models  
   └─ PlaNet → Dreamer  
        ↓  
03 Transformer for Control  
   └─ Decision Transformer → Trajectory Transformer → CQL  
        ↓  
04 Generative Policies (Diffusion)  
   └─ Diffusion Policy → 3D Diffuser Actor → 3D Diffusion Policy  
        ↓  
05 Language + Object-Centric Policies  
   └─ Language-Guided DP → Collision-Aware Diffusion  
        ↓  
06 Foundation Robot Models (VLA)  
   └─ RoboCat → RT-1 → RT-2 → RT-X / OpenVLA / Octo
```
