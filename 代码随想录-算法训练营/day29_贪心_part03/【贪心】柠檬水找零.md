# 题目：柠檬水找零 (Lemonade Change) 🍋💵

## 题目链接  
https://leetcode.cn/problems/lemonade-change/

---

## 🧠 解题思路

### 📋 题意理解  
在柠檬水摊上：

- 每杯柠檬水售价 **5 元**
- 顾客依次用 `5 / 10 / 20` 面额付款
- 初始时 **手中没有任何零钱**
- 每位顾客来时，必须当场找零  
- 判断是否 **能为所有顾客成功找零**

---

## 🌟 核心思想：贪心（Greedy）

这道题的贪心策略非常经典：

> **优先保留 5 元零钱，10 元次之**

原因：

- 5 元是最“通用”的零钱
- 10 元只能配合 5 元给 20 元找零
- 20 元对找零没有任何帮助

因此，在给 20 找零时：

- **优先使用 10 + 5**
- 如果没有 10，再使用 **3 个 5**

---

## ✨ 算法策略拆解

维护两个变量：

```cpp
int five = 0; // 当前拥有的 5 元数量
int ten  = 0; // 当前拥有的 10 元数量
```

依次处理每一位顾客：

---

### ✅ 情况一：顾客付 5 元

- 不需要找零
- `five++`

---

### ✅ 情况二：顾客付 10 元

- 需要找 5 元
- 如果 `five == 0` → 无法找零，直接失败
- 否则：
  - `five--`
  - `ten++`

---

### ✅ 情况三：顾客付 20 元

需要找零 15 元，有两种方式：

1. **10 + 5（最优）**
2. **5 + 5 + 5**

贪心选择：

- 如果有 `ten > 0 && five > 0`：
  - 优先用 `10 + 5`
- 否则：
  - 必须有 `five >= 3`
- 两种都不满足 → 返回 `false`

---

## 💻 代码实现

```cpp
class Solution {
public:
    bool lemonadeChange(vector<int>& bills) {
        int five = 0, ten = 0;
        for (int i = 0; i < bills.size(); i++){
            if (bills[i] == 5)
                five++;
            else if (bills[i] == 10){
                if (five == 0)
                    return false;
                five--; ten++;
            }
            else{
                if ((ten < 1 && five < 3) || five < 1)
                    return false;
                if (ten > 0){
                    ten--; five--;
                }
                else
                    five -= 3;
            }
        }
        return true;
    }
};
```

---

## 🔍 关键点解析

### ❓ 为什么给 20 时要优先用 10 + 5？

假设有：

```
five = 3, ten = 1
```

如果用 `3 个 5`：

- five = 0, ten = 1  
后面如果再来一个 10，就无法找零 ❌

如果用 `10 + 5`：

- five = 2, ten = 0  
后面还能继续找零 ✅

👉 **10 元的使用价值低于 5 元**

## 🧠 思维模型总结

- 问题本质是：**零钱的可持续性**
- 贪心的核心不是“眼前能不能找”，而是：
  > **为未来保留更灵活的零钱组合**
