# 并查集进阶：处理有向图中存在多个父节点的情况（Leetcode 685）

---

## 🧠 问题背景

在有向图中，每个节点最多应该只有一个父节点。  
但现在给定的图中包含了 `n` 条边，且节点数为 `n`，  
说明其中 **必然有一条边是多余的**。

图中可能出现以下两种异常：

1. **一个节点有两个入边**
2. **图中存在环**

---

## 🔧 解法核心：并查集 + 入边记录

我们需要结合两部分信息：

- 使用并查集判断图中是否形成了环
- 使用一个数组记录每个节点的“入边来源”，检测是否有入度为 2 的节点

---

## 🔍 特判情况说明

- 如果某个节点 `t` 被多个 `s` 指向，即 `t` 有两个父节点，记录这个异常节点 `flag`
- 如果图中有环，使用并查集判断出导致成环的那一条边

我们需要综合这两种信息决定最后要删掉哪条边。

---

## 📦 代码实现

```cpp
#include <cstdio>
#include <iostream>
#include <vector>
using namespace std;

int find(int u, auto &father){
    if (father[u] == u)
        return u;
    return father[u] = find(father[u], father);
}

bool isSame(int u, int v, auto &father){
    return (find(u, father) == find(v, father));
}

void joint(int u, int v, auto &father){
    u = find(u, father);
    v = find(v, father);
    if (u == v)
        return; 
    father[u] = v;
}

int main(){
    int N;
    cin >> N;

    // 记录每个节点的入边来源（可能有多个）
    vector<vector<int>> NodeIN(N + 1, vector<int>());

    // 并查集初始化
    vector<int> father(N + 1, 0);
    for (int i = 1; i <= N; i++){
        father[i] = i;
    }

    int ans1, ans2;
    int flag = -1; // 多个父节点的异常点

    for (int i = 0; i < N; i++){
        int s, t;
        cin >> s >> t;

        // 入边记录
        NodeIN[t].push_back(s);
        if (NodeIN[t].size() > 1)
            flag = t;

        // 判断是否形成环
        if (isSame(s, t, father)){
            ans1 = s;
            ans2 = t;
        }

        // 合并两个集合
        joint(s, t, father);
    }

    // 情况：出现一个点有两个父节点，需要根据是否成环判断要删哪条边
    if (flag != -1){
        for (int i = NodeIN[flag].size() - 1; i >= 0; i--){
            int cnt = NodeIN[flag][i];
            if (isSame(cnt, flag, father) && !NodeIN[cnt].empty()){
                ans1 = cnt;
                ans2 = flag;
                break;
            }
        }
    }

    cout << ans1 << ' ' << ans2 << endl;
    return 0;
}
```

---

## ✅ 总结逻辑

- 对于所有边，尝试逐步添加到图中
- 如果某一条边让两个点连通，说明形成了 **环**
- 如果某节点有两个父节点，说明违反有向图规则
- 最终根据这两种异常结合情况选择删掉哪一条边

---

## 📌 示例输入输出

```text
输入：
5
1 2
2 3
3 4
4 1
1 5

输出：
4 1
```

说明：删除边 `4 -> 1` 可以让图重新变成合法的树结构。
