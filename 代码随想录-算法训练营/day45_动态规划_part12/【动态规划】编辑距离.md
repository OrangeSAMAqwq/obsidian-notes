# ✍️ 题目：编辑距离（Edit Distance / Levenshtein Distance）🧠✂️➕

---

## 🧩 题目理解

给定两个字符串：

- `word1`
- `word2`

每次可以执行 **三种操作之一**：

1. **插入一个字符**
2. **删除一个字符**
3. **替换一个字符**

目标是：

> 使用最少的操作次数，将 `word1` 转换成 `word2`。

---

## 💡 解题思路：经典字符串 DP

这是字符串 DP 中**最经典、最重要的一道母题**，几乎所有字符串编辑类问题都能从它演化出来。

---

## 📐 状态定义

```text
dp[i][j]：
表示将 word1 的前 i 个字符
转换为 word2 的前 j 个字符
所需要的最少操作次数
```

其中：

- `i ∈ [0, m]`
- `j ∈ [0, n]`

---

## 🧱 初始化（非常关键）

### 1️⃣ word2 为空

```text
dp[i][0] = i
```

含义：  
把 `word1[0..i)` 变成空串，只能删除 `i` 个字符。

---

### 2️⃣ word1 为空

```text
dp[0][j] = j
```

含义：  
把空串变成 `word2[0..j)`，只能插入 `j` 个字符。

---

## 🔁 状态转移方程（核心）

考虑当前字符：

- `word1[i-1]`
- `word2[j-1]`

---

### ✅ 情况一：字符相等

```text
word1[i-1] == word2[j-1]
```

无需任何操作，直接继承之前状态：

```text
dp[i][j] = dp[i-1][j-1]
```

---

### ❌ 情况二：字符不相等

```text
word1[i-1] != word2[j-1]
```

此时可以进行 **三种操作**，取最小值：

#### ① 替换字符
```text
dp[i-1][j-1] + 1
```

#### ② 删除 word1[i-1]
```text
dp[i-1][j] + 1
```

#### ③ 插入一个字符到 word1
```text
dp[i][j-1] + 1
```

综合：

```text
dp[i][j] = min(
    dp[i-1][j-1],
    dp[i-1][j],
    dp[i][j-1]
) + 1
```

---

## 💻 C++ 实现代码（你的版本）

```cpp
class Solution {
public:
    int minDistance(string word1, string word2) {
        int m = word1.size(), n = word2.size();
        vector<vector<int>> dp(m + 1, vector<int>(n + 1, 0));

        // 初始化
        for (int i = 0; i <= m; i++) {
            dp[i][0] = i;
        }
        for (int j = 0; j <= n; j++) {
            dp[0][j] = j;
        }

        // 状态转移
        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (word1[i - 1] == word2[j - 1]) {
                    dp[i][j] = dp[i - 1][j - 1];
                } else {
                    dp[i][j] = min(
                        dp[i - 1][j - 1],
                        min(dp[i - 1][j], dp[i][j - 1])
                    ) + 1;
                }
            }
        }
        return dp[m][n];
    }
};
```

---

## 📊 复杂度分析

- **时间复杂度**：`O(m × n)`
- **空间复杂度**：`O(m × n)`
  - 可优化为一维 DP（滚动数组）

## 🧠 记忆要点 ⭐（非常重要）

```text
编辑距离 = 插入 + 删除 + 替换
```

```text
三方向来源：
↖ 替换
↑ 删除
← 插入
```

---

## 🔗 和其它字符串 DP 的关系

| 题目 | 操作类型 |
|----|----|
| 编辑距离 | 插入 / 删除 / 替换 |
| 删除操作得到相同字符串 | 只允许删除 |
| 不同的子序列 | 方案数 |
| LCS | 最大公共长度 |

📌 **编辑距离是字符串 DP 的“祖师爷”**，  
其它题目基本都是在它的操作集合上做减法或变形。

---

## ✅ 总结一句话

> **编辑距离问题的本质：**
>  
> 把「当前字符是否相等」转化为  
> 「三种操作取最小代价」的问题。
